# Consultative GroupChat Planning Artifacts

This directory houses all inputs, outputs, and metadata files generated by the **GroupChat Planning Protocol**.

## Directory Structure

```
PLANNING/
├── README.md                           # This documentation
├── groupchat_request_<sessionId>.json  # Fallback request file / input trigger
├── groupchat_<sessionId>.json          # Structured JSON session logs (Schema v1)
├── groupchat_<sessionId>.md            # Human-readable ordered markdown transcript
└── draft_<sessionId>.md                # Clean final plan from 小C (Codex)
```

## The 7-Step Consultative Protocol

Every planning session progresses through a structured 7-step conversation between the three agents:
1. **`codex_proposal`** (小C): Proposes the initial plan based on the input objective.
2. **`grok_advice`** (小B): Provides feedback, risks, and review criteria.
3. **`antigravity_advice`** (小A): Provides execution constraints and technical details.
4. **`codex_response`** (小C): Explicitly summarizes which 小A/小B suggestions were accepted, modified, or rejected.
5. **`antigravity_final`** (小A): Advises final alignment (e.g. `小A全面認可`).
6. **`grok_final`** (小B): Advises final alignment (e.g. `小B全面認可`).
7. **`codex_closing`** (小C): Closes the session and presents the final clean plan.

## Runner Usage

The groupchat planning runner can be executed using the following npm command:
```bash
npm run groupchat:plan -- --session <sessionId> --input "Your objective here"
```

### Options
- `--session <id>`: (Required) Unique 3-digit session ID (e.g., `001`).
- `--input <text>`: Objective/requirements text.
- `--input-file <path>`: Path to a file containing the objective/requirements.
- `--task <taskNum>`: (Optional) ID of a formal task to link with the planning session.
- `--force`: (Optional) Overwrite existing output files instead of failing.
