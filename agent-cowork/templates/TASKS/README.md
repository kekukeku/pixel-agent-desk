# Task Specifications

This directory holds task specification documents for the repository.

Each task document is named `task_NNN.md` (where `NNN` is a 3-digit zero-padded index, e.g. `001`, `002`) and tracks:
- **Status**: `DRAFT`, `IN_PROGRESS`, `UNDER_REVIEW`, `APPROVED`, `MERGED`, `REJECTED`
- **Objective**: Description of what needs to be implemented
- **Acceptace Criteria**: Checklists for validation
- **Implementation Notes**: Specific planning decisions made by the team

Task files are authored and updated by Codex (Layer 1 Planner).
