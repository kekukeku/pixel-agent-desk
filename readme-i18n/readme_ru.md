## О Хранителях в Машине

В старину ни один мастер не касался пера без незримого хранителя за плечом. Век сменил светильники: пергамент стал стеклом экрана, а духи облачились в пиксели и двоичный код, но их верная служба продолжается.
*Pixel Agent Desk* дарует этим славным часовым тихую двухмерную обитель — скромный офис, где ваши ИИ-агенты думают, трудятся и мирно дремлют в перерывах.
Приоткройте завесу, сделайте невидимое видимым и позвольте древней магии отладки твориться на ваших глазах.

*[Читать полную прелюдию](docs/readme-prelude.md) — О Хранителях в Машине*

Pixel Agent Desk — это автономное приложение на Electron, которое отслеживает события жизненного цикла агентов и отображает активные ИИ-сессии в виде анимированных пиксельных персонажей в 2D-офисе. Из коробки поддерживается пять основных рабочих пространств агентов:

- **Claude Cowork**
- **Codex**
- **Grok Build**
- **Antigravity**
- **OpenWork**

Приложение является слоем наблюдения и визуализации. Оно не распределяет работу, не назначает задачи и не управляет вашими агентами.

## Основные возможности

- **Автономный наблюдатель** — PAD работает независимо в качестве наблюдателя для GUI и TUI рабочих пространств агентов.
- **Пиксельный офис** — 2D виртуальный офис, где активные агенты появляются в виде анимированных пиксельных персонажей, управляемых событиями жизненного цикла.
- **Системный ростер** — Карточки панели управления в реальном времени, отображающие состояние агента, активные инструменты, источники, использование токенов и измеряемую стоимость при наличии.
- **Пять дополнительных интеграций** — Claude Cowork, Codex, Grok Build, Antigravity и OpenWork, с совместимостью с OpenCode через ядро OpenWork.
- **Аналитика токенов и стоимости** — Показывает видимость токенов для поддерживаемых агентов, кроме Antigravity, и оценивает стоимость только при наличии надёжных данных о ценах.
- **Сетка активности и обзор GroupChat** — Доступ к историческим воспроизведениям сессий и визуальным тепловым картам активности.
- **Универсальный API событий** — Пользовательские внешние инструменты могут отправлять нормализованные события через `POST /events/agent`.
- **Автоматическое восстановление** — Безопасно восстанавливает активные сессии агентов при перезапуске приложения с использованием проверенных PID или конфигураций допуска.

## Требования

**Для запуска Pixel Agent Desk:**
- **macOS (рекомендуется):** отдельная установка Node не требуется — [`Install.command`](Install.command) загружает переносимый Node.js 22 в `~/.local/node` при первом запуске.
- **Windows / Linux / ручной macOS:** **Node.js** 20 или новее и **npm**
- **macOS, Windows или Linux**

*Примечание: Рабочие пространства агентов **не** являются обязательными для запуска приложения. Pixel Agent Desk работает как независимый наблюдатель. Отсутствующие платформы будут отражены в диагностике, но никогда не вызовут сбой или блокировку панели управления.*

## Быстрый старт

### macOS — Запуск с рабочего стола (рекомендуется)

1. **Первоначальная настройка**: Дважды щёлкните [`Install.command`](Install.command) в корне репозитория.
   - Загружает официальные бинарники Node.js в `~/.local/node`, если у вас ещё нет Node 20+.
   - Выполняет `npm install` для зависимостей Pixel Agent Desk.
   - Требуется доступ к сети при первом запуске.
2. **Запуск панели управления**: Дважды щёлкните [`Start.command`](Start.command).
   - Использует тот же Node.js (`~/.local/node` или существующий системный Node 20+).
   - Открывает окно панели управления через `npm start`.
   - *Примечание о правах: Если macOS сообщает, что `Install.command` или `Start.command` не удаётся открыть, выполните `chmod +x Install.command Start.command` в этой папке через Терминал.*
   - *Примечание Gatekeeper: Если macOS блокирует выполнение, щёлкните правой кнопкой по файлу `.command` и выберите **Открыть**.*

### Все платформы — Запуск из исходников

Для ручного клонирования и запуска из исходников:

```bash
git clone https://github.com/kekukeku/pixel-agent-desk.git
cd pixel-agent-desk
npm install
npm start
```

При запуске:
- Открывается окно панели управления Pixel Agent Desk (динамически отображается `Офис {username}`, соответствующий профилю вашей учётной записи ОС).
- Локальный сервер шлюза событий начинает прослушивание на `127.0.0.1:47821`.
- Настроенные наблюдатели и интеграции пересылки регистрируются и готовятся к приёму событий агентов.

### Диагностика

Чтобы проверить состояние обнаружения локальных интеграций агентов без написания каких-либо конфигурационных хуков или запуска наблюдателей:

```bash
npm run diagnose:integrations
```

## Представления панели управления

Боковая навигация предоставляет четыре основных режима просмотра для мониторинга и изучения ваших сессий агентов:

| Представление | Назначение | Подробности |
|---|---|---|
| **Overview** | Основной 2D-холст офиса и Живой ростер | Просмотр анимированных пиксельных спрайтов, движущихся и работающих, наряду с картами статуса агентов в реальном времени. Поддержка окна PiP (Картинка в картинке). |
| **Activity Mesh** | Интерактивная тепловая матрица | Отображает ежедневную/почасовую частоту событий и пики. |
| **GroupChat Review** | Локальное воспроизведение сессии | Воспроизводит записанные многоагентные обсуждения (`groupchat_*.json`) прямо на визуальном 2D-холсте офиса. |
| **Metered API Usage** | Панель использования токенов и биллинга | Отображает количество токенов для поддерживаемых агентов, оценочную стоимость при надёжных ценах и пиковое использование контекстного окна (CTX%) для Grok Build. |

## Интеграции

| Агент | Механизм | Путь конфигурации / данных | Записывает конфигурацию? | Примечания |
|---|---|---|---|---|
| Claude Cowork | Пересылчик событий | `~/.claude/settings.json` | Да | Автоматически регистрирует хуки, принадлежащие PAD; мигрирует унаследованные HTTP-хуки при наличии |
| Codex | Наблюдатель JSONL только для чтения | `~/.codex/` | Нет | Сканирует файлы сессий примерно каждые 2 секунды |
| Grok Build | Пересылчик событий + наблюдатель | `~/.grok/hooks/pixel-agent-desk.json` + `~/.grok/sessions/**/signals.json` | Да | Хук управляет жизненным циклом; наблюдатель отслеживает токены и CTX% |
| Antigravity | Пересылчик событий | `~/.gemini/config/hooks.json` | Да | Непосредственно интегрирует исполняемый файл пересылчика |
| OpenWork / OpenCode | Плагин, совместимый с OpenCode | `~/.config/opencode/plugins/pad-adapter.js` | Да | OpenWork поддерживается через своё ядро, совместимое с OpenCode |

В упакованных сборках вспомогательные файлы материализуются в `~/.pixel-agent-desk/runtime/` для запуска пересылчиков через двоичный файл Electron с использованием `ELECTRON_RUN_AS_NODE=1`. В режиме разработки из исходников пересылчики запускаются непосредственно из папки с исходным кодом репозитория.

См. [docs/integration-smoke-test.md](docs/integration-smoke-test.md) для подробного руководства по интеграционному тестированию.

*Важное примечание: Если нет активных агентов, **пустой виртуальный офис** — это нормально и не означает, что PAD работает со сбоем. Анимированные персонажи появляются только после того, как их агент отправит хотя бы одно событие (например, откроет поддерживаемое рабочее пространство или отправит запрос).*

Чтобы отключить интеграции Pixel Agent Desk, удалите только принадлежащие PAD хуки/плагины или ключи конфигурации:

| Агент | Что удалить |
|---|---|
| Claude Cowork | Удалите принадлежащие PAD записи хуков из `~/.claude/settings.json` |
| Grok Build | Удалите `~/.grok/hooks/pixel-agent-desk.json` |
| Antigravity | Удалите ключ `"pixel-agent-desk"` из `~/.gemini/config/hooks.json` |
| OpenWork / OpenCode | Удалите `~/.config/opencode/plugins/pad-adapter.js` |
| Codex | Конфигурация не записывается — просто завершите PAD для отключения |

Дополнительный кэш (безопасно удалить; PAD пересоздаст его при следующем запуске):

```text
~/.pixel-agent-desk/runtime/
```

Перезапустите затронутое рабочее пространство агента после изменения, чтобы перезагрузить конфигурации.

## Конфигурация

Pixel Agent Desk считывает дополнительную пользовательскую конфигурацию из:

```text
~/.pixel-agent-desk/config.json
```

Пример:

```json
{
  "integrations": {
    "claude": {
      "enabled": true
    },
    "opencode": {
      "enabled": true
    }
  }
}
```

Текущие конфигурационные ворота:

- `integrations.claude.enabled: false` пропускает регистрацию хука Claude Cowork и сканирование стенограмм.
- `integrations.opencode.enabled: false` пропускает регистрацию плагина OpenCode.

Другие интеграции обнаруживаются по возможностям и безопасно открываются, если их платформа не установлена.

## Нормализованный API событий агента

Пользовательские инструменты могут сообщать об активности, отправляя нормализованные события на:

```text
POST http://127.0.0.1:47821/events/agent
Content-Type: application/json
```

Пример:

```json
{
  "event": "agent.working",
  "agent_id": "custom-session-1",
  "source": "my-custom-agent",
  "name": "Research Agent",
  "project_path": "/path/to/project",
  "model": "gpt-4o",
  "tool": "Bash",
  "parent_id": null,
  "pid": 12345,
  "timestamp": 1781550497208,
  "token_usage": {
    "input_tokens": 1200,
    "cached_input_tokens": 500,
    "output_tokens": 400
  },
  "context_usage": {
    "kind": "snapshot",
    "tokens_used": 50000,
    "window_tokens": 200000,
    "percent": 25
  },
  "metadata": {}
}
```

### Поддерживаемые события

- `agent.started` — Регистрирует или обновляет сессию агента.
- `agent.thinking` — Показывает состояние размышления и может накапливать использование токенов.
- `agent.working` — Показывает состояние работы и активный инструмент.
- `agent.idle` — Показывает состояние отдыха/простоя.
- `agent.done` — Отмечает завершённое действие.
- `agent.error` — Показывает состояние ошибки.
- `agent.help` — Показывает состояние разрешения/помощи.
- `agent.removed` — Удаляет персонажа из офиса.

## Восстановление сессии и отображаемые имена

Pixel Agent Desk сохраняет активные сессии и пытается восстановить их при перезапуске, когда источник можно безопасно проверить.

Дополнительные локальные файлы сопоставления:

- `~/.pixel-agent-desk/name-map.json` сопоставляет стабильные идентификаторы сессий с отображаемыми именами.
- `~/.pixel-agent-desk/watcher-allowlist.json` — устаревшее имя файла, используемое как список допуска для восстановления пользовательских/ручных сессий. Не связано с удалённым Python-наблюдателем.

Пример `name-map.json`:

```json
{
  "codex-main": "Codex",
  "antigravity-ui": "Antigravity"
}
```

## Настройка аватара

Выбор аватаров хранится локально в браузерном хранилище:

```text
Ключ localStorage: pixel-agent-desk.avatarOverrides.v1
```

Значение сопоставляет стабильные идентификаторы агентов с индексами аватаров. Выбор "Сбросить по умолчанию" удаляет переопределение.

## Отображение токенов и стоимости

Pixel Agent Desk отображает использование ресурсов в зависимости от данных, предоставляемых агентом:

- **Агенты с видимостью токенов**: Claude Cowork, Codex, Grok Build и OpenWork/OpenCode могут отображать использование токенов, когда их локальные данные событий или сессий его раскрывают.
- **Агенты, учитывающие стоимость**: Когда использование токенов можно сопоставить с надёжным тарифом в [src/pricing.js](src/pricing.js), Pixel Agent Desk оценивает стоимость. В противном случае отображается только использование без выдуманных цифр биллинга.
- **Агенты, учитывающие контекст (например, Grok Build)**: Отображает пиковый процент контекстного окна (`CTX: N tok` или процентное давление). Значения снимков контекста не накапливаются. Ежедневная тепловая карта записывает пиковые контекстные токены за день.
- **Antigravity**: Поддерживается видимость жизненного цикла, но обнаружение токенов в настоящее время недоступно.

См. [docs/integration-smoke-test.md](docs/integration-smoke-test.md) §5.3 для проверки Grok CTX.

*Примечание: Убедитесь, что `npm start` закрыт при проверке упакованных хуков, поскольку только один экземпляр PAD может привязываться к порту локального сервера событий (`47821`).*

## Журнал отладки

Pixel Agent Desk записывает журналы выполнения в `debug.log`:

- **Из исходников (`npm start`)**: `src/debug.log` внутри клонированного репозитория
- **Упакованное приложение (macOS)**: `~/Library/Application Support/pixel-agent-desk/debug.log`
- **Упакованное приложение (Windows)**: `%APPDATA%/pixel-agent-desk/debug.log`
- **Упакованное приложение (Linux)**: `~/.config/pixel-agent-desk/debug.log`

Ищите строки `[Processor]` и `[Event]` при проверке, что события агентов достигают офиса.

## Устранение неполадок

| Симптом | Вероятная причина | Решение |
|---|---|---|
| Персонажи не появляются | Событие агента ещё не достигло PAD | Запустите сессию агента один раз, затем проверьте `debug.log` (см. Журнал отладки выше) на наличие строк `[Processor]` |
| Офис пуст (нет персонажей) | Нормальное состояние при запуске или неактивных сессиях | Анимированные персонажи появляются только после того, как их агенты отправят хотя бы одно событие (например, откроют поддерживаемое рабочее пространство или отправят запрос). Подтвердите наличие событий `[Processor]` в `debug.log`. |
| Диагностика сообщает Codex `active=false` | Диагностика доступна только для чтения и не запускает наблюдателей | Используйте `npm start`; Codex должен стать активным, если установлен |
| Grok или Antigravity не отображаются в упакованном приложении | Команда хука всё ещё указывает на старый путь к исходникам | Перезапустите упакованное приложение, чтобы хуки обновились; проверьте конфигурацию хука для `~/.pixel-agent-desk/runtime/forwarders/` |
| Команда хука использует `node` в упакованной валидации | Конфигурация хука была сгенерирована dev-приложением или старой версией | Закройте dev PAD, откройте упакованное `.app`, затем перепроверьте конфигурацию хука |
| OpenCode не отображается | Плагин не установлен или OpenCode его не загрузил | Проверьте `~/.config/opencode/plugins/pad-adapter.js`, затем перезапустите OpenCode/OpenWork |
| Claude Cowork не отображается | Отсутствуют или отключены хуки Claude Cowork | Выполните `npm run diagnose:integrations` и проверьте `~/.claude/settings.json` |
| Устаревший персонаж остаётся | Восстановление сохранённой сессии всё ещё имеет совпадающий ID | Удалите устаревшие записи из `name-map.json` или `watcher-allowlist.json`, затем перезапустите |
